## 🎯 创作前准备

在开始创作之前，了解 Hugo-Teek 的内容组织方式和创作规范，能让你的写作更加高效。

### 内容存储位置

::: code-group

``` [文档目录📁]
hugo-teek-site/content/
├── docs/               # 文档类内容
│   ├── hugo-teek默认文档/
│   └── 专题/
├── posts/              # 博客文章
├── about/              # 关于页面
└── _settings/          # 设置页面
```

``` [文章存放建议📝]
技术文档 → content/docs/分类名/
博客随笔 → content/posts/
个人介绍 → content/about/
```

:::

## 📋 Front Matter 配置

每篇文章的开头都需要配置 Front Matter，用于定义文章的元数据。

### 基础配置模板

::: code-group

```yaml [简洁模板✨]
---
title: 文章标题
date: "2026-03-02T10:00:00+08:00"
description: "文章描述"
tags:
    - 标签1
    - 标签2
categories:
    - 分类名
draft: false
---
```

```yaml [完整模板📋]
---
title: 文章标题                          # 文章标题
date: "2026-03-02T10:00:00+08:00"       # 发布时间
description: "文章描述"                  # SEO描述
coverImg: https://example.com/img.jpg   # 封面图片
url: /custom-path                        # 自定义URL
weight: 1                                # 排序权重
tags:                                    # 标签列表
    - 标签1
    - 标签2
categories:                              # 分类列表
    - 分类名
draft: false                             # 草稿状态
featured: true                           # 首页推荐
---
```

:::

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| title | ✅ | 文章标题，显示在列表和详情页 |
| date | ✅ | 发布时间，ISO 8601 格式 |
| description | ❌ | 文章描述，用于SEO和列表展示 |
| coverImg | ❌ | 封面图片URL |
| url | ❌ | 自定义访问路径 |
| weight | ❌ | 排序权重，数字越小越靠前 |
| tags | ❌ | 文章标签，支持多个 |
| categories | ❌ | 文章分类，支持多个 |
| draft | ❌ | 草稿状态，true为草稿不发布 |
| featured | ❌ | 推荐状态，true会显示在首页 |

## 🎨 内容组织技巧

### 文章结构设计

::: code-group

```markdown [技术文档结构🔧]
# 文章标题

## 简介
简要介绍文章主题

## 环境准备
需要的工具和依赖

## 详细步骤
1. 步骤一
2. 步骤二
3. 步骤三

## 常见问题
Q1: 问题描述？
A: 解决方案

## 总结
总结要点
```

```markdown [博客文章结构✍️]
# 文章标题

## 背景
为什么写这篇文章

## 正文内容
详细叙述...

## 心得体会
个人感悟...

## 参考资料
相关链接...
```

:::

### 目录层级建议

::: info 📐 层级使用规范
- 文章主标题使用 H1（#）
- 大章节使用 H2（##）
- 小章节使用 H3（###）
- 避免使用超过 H4（####）
- 保持层级结构清晰，不要跳级
:::

## 🏷️ 标签与分类管理

### 标签使用建议

::: code-group

```yaml [技术类标签💻]
tags:
    - hugo
    - 前端开发
    - 静态网站
    - 教程
```

```yaml [生活类标签🌟]
tags:
    - 随笔
    - 读书笔记
    - 生活感悟
```

```yaml [项目类标签📁]
tags:
    - 开源项目
    - 工具推荐
    - 使用心得
```

:::

### 分类规划建议

::: tip 🗂️ 分类设计原则
1. **分类要少而精** - 建议3-5个主要分类
2. **分类要互斥** - 一篇文章通常只属于一个分类
3. **标签要丰富** - 一篇文章可以有多个标签
4. **命名要统一** - 使用统一的命名规范
:::

## 🔍 SEO 优化建议

### 基础SEO配置

::: code-group

```yaml [文章级SEO🔍]
---
title: "包含关键词的标题"
description: "150字以内的描述，包含核心关键词"
url: /seo-friendly-url
---
```

```markdown [内容级SEO✍️]
# 文章标题包含核心关键词

## 第一段要包含关键词
文章开头100字内出现核心关键词...

## 合理使用小标题
使用H2、H3标签组织内容结构...

## 添加内部链接
参考[相关文章](/other-post/)...

## 图片添加Alt属性
![关键词描述](image.jpg)
```

:::

### URL 设计规范

::: tip 🔗 URL优化建议
- 使用小写字母
- 使用短横线分隔单词
- 包含核心关键词
- 避免使用数字ID
- 保持简洁有意义

**示例：**
- ✅ `/hugo-teek-tutorial`
- ✅ `/frontend-development-guide`
- ❌ `/post-12345`
- ❌ `/2026/03/02/abc`
:::

## 💡 写作技巧

### 提升可读性

::: code-group

```markdown [段落优化📖]
❌ 避免：
一大段文字没有分段，读起来很累...

✅ 推荐：
每段控制在3-5行。

适当分段让阅读更轻松。

使用空行分隔不同主题。
```

```markdown [列表使用📋]
❌ 避免：
第一点是什么，第二点是什么，第三点是什么...

✅ 推荐：
- 第一点说明
- 第二点说明
- 第三点说明
```

```markdown [强调重点🎯]
使用 **加粗** 突出重点
使用 `代码` 标记技术术语
使用 > 引用重要内容
```

:::

### 代码展示规范

::: info 💻 代码块最佳实践
1. **指定语言** - 让代码高亮生效
2. **添加标题** - 说明代码用途
3. **适当注释** - 解释关键逻辑
4. **控制长度** - 过长代码考虑分段
5. **提供说明** - 代码前后添加解释
:::

## 📊 内容统计与分析

### 文章数据统计

Hugo-Teek 内置不蒜子统计，可查看文章阅读量：

```toml
# config/teek-plugins/services.toml
[services.busuanzi]
    enabled = true
```

### 热门文章推荐

使用 `featured: true` 标记重要文章，它们会显示在首页推荐区域。

::: tip 🌟 推荐文章选择标准
- 内容质量高
- 阅读量较高
- 具有代表性
- 时效性较强
:::

## 🛠️ 常见问题

### 内容相关问题

::: code-group

```markdown [文章不显示❓]
检查：
1. draft 是否为 false
2. date 是否为未来时间
3. 文件是否在正确的 content 目录下
```

```markdown [排序不对❓]
解决：
1. 检查 weight 字段设置
2. weight 数字越小越靠前
3. 相同weight按时间排序
```

```markdown [标签不生效❓]
注意：
1. tags 使用数组格式
2. 注意缩进使用空格
3. 避免特殊字符
```

:::

## 📚 进阶技巧

### 自定义模板使用

Hugo-Teek 提供了多个实用的短代码，方便在文章中插入复杂内容：

::: tip 🎯 alert 短代码
用于创建提示框，支持多种类型：

```
{{</* alert type="tip" title="提示" */>}}
自定义提示内容
{{</* /alert */>}}
```

**支持的类型：** `tip`、`warning`、`danger`、`info`、`note`、`important`、`caution`、`details`
:::

::: tip 🎨 custom-block 短代码
功能与 alert 类似，参数传递方式不同：

```
{{</* custom-block type="warning" title="注意" */>}}
这里是警告内容
{{</* /custom-block */>}}
```
:::

::: tip 🎬 video 短代码
用于嵌入视频内容：

```
{{< video src="https://example.com/video.mp4" >}}
```

**支持的平台：** bilibili、youtube、tencent、youku、vimeo、xigua

```
{{< video provider="bilibili" id="BV1xx411c7mD" >}}
```
:::

### 多语言内容

::: info 🌍 多语言支持
Hugo-Teek 支持多语言内容创作：
- 在 `content/` 下创建语言目录，如 `content/en/`
- 配置 `languages.toml` 启用多语言
- 使用 `{{ i18n "key" }}` 引用翻译
:::
