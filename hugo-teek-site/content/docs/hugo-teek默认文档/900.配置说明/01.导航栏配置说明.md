## 🧭 导航栏配置概述

Hugo-Teek 主题采用灵活的菜单配置系统，支持多级下拉菜单、图标配置、外部链接等多种功能。导航栏配置采用 TOML 格式，通过 Hugo 原生的菜单系统进行管理。

### ✨ 导航栏特性

::: tip 🎨 设计特性
- 多级下拉菜单：支持无限层级嵌套
- 图标支持：内置 Bootstrap Icons 图标库
- 外部链接：支持新窗口打开外部链接
- 响应式布局：移动端自动折叠为汉堡菜单
:::

::: info ⚙️ 配置特性
- 权重排序：通过 weight 控制菜单顺序
- 标识符系统：使用 identifier 建立层级关系
- 参数扩展：支持自定义参数扩展功能
- 主题隔离：不同主题可拥有独立菜单配置
:::

## 📁 配置文件路径

Hugo-Teek 采用分层配置结构，菜单配置文件位于以下位置：

::: code-group

```toml [通用菜单配置📄]
# hugo-teek-site/config/teek-plugins/menus.toml
# 所有主题共用的菜单配置
# 推荐在此文件中进行菜单配置
```

```toml [主题特定菜单📄]
# hugo-teek-site/config/themes/hugo-teek/menus.toml
# 仅当前主题使用的菜单配置
# 用于覆盖通用配置
```

:::

### 配置优先级

```
配置合并顺序（优先级从低到高）：
1️⃣ config/themes/{theme}/menus.toml（主题默认配置）
2️⃣ config/teek-plugins/menus.toml（通用配置）
3️⃣ 运行时通过管理后台修改的配置
```

## 📝 基础菜单配置

### 单级菜单项

最简单的菜单配置，只包含名称、链接和排序权重：

::: code-group

```toml [基础菜单项📝]
[[main]]
  name = "🏡 首页"
  url = "/"
  weight = 1
```

```toml [配置说明📝]
[[main]]              # ◀️ 菜单组标识，固定为 main
  name = "🏡 首页"     # ◀️ 显示名称，支持 emoji
  url = "/"           # ◀️ 链接地址，支持相对/绝对路径
  weight = 1          # ◀️ 排序权重，数字越小越靠前
```

:::

### 菜单配置字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 菜单显示名称，支持 emoji 图标 |
| `url` | ✅ | 链接地址，相对路径或绝对 URL |
| `weight` | ✅ | 排序权重，数字越小越靠前 |
| `identifier` | ❌ | 唯一标识符，用于建立父子关系 |
| `parent` | ❌ | 父菜单的 identifier，用于创建子菜单 |

## 🌳 多级下拉菜单

### 创建下拉菜单结构

通过 `identifier` 和 `parent` 字段建立层级关系：

::: code-group

```toml [完整多级菜单🌳]
# 一级菜单（父菜单）
[[main]]
  identifier = "tech"      # ◀️ 唯一标识符
  name = "📚 技术分类"
  url = "#"                # ◀️ # 表示纯下拉，不跳转
  weight = 2

# 二级菜单（子菜单）
[[main]]
  name = "🖥️ 运维"
  parent = "tech"          # ◀️ 对应父菜单的 identifier
  url = "/linux/"
  weight = 1

[[main]]
  name = "🎨 前端"
  parent = "tech"
  url = "/frontend/"
  weight = 2
```

```toml [层级关系说明📝]
📚 技术分类 (identifier: tech)
├── 🖥️ 运维 (parent: tech)
├── 🎨 前端 (parent: tech)
└── 💻 编程 (parent: tech)
```

:::

### 多级菜单最佳实践

::: warning ⚠️ 注意事项
- 父菜单的 `identifier` 必须唯一，不可重复
- 子菜单的 `parent` 值必须与父菜单的 `identifier` 完全匹配
- 父菜单设置 `url = "#"` 表示纯下拉菜单，点击不跳转
- 建议合理设置 `weight`，保持菜单逻辑顺序
:::

## 🎨 图标配置

### 使用 Bootstrap Icons

Hugo-Teek 内置 Bootstrap Icons 图标库，可通过 `params.icon` 配置菜单图标：

::: code-group

```toml [带图标的菜单🎨]
[[main]]
  name = "🖥️ 运维"
  parent = "tech"
  url = "/linux/"
  weight = 1
  [main.params]
    icon = "monitor"     # ◀️ Bootstrap Icons 名称
```

```toml [常用图标示例🎨]
[main.params]
  icon = "house"        # ◀️ 首页图标
  icon = "book"         # ◀️ 文档图标
  icon = "code"         # ◀️ 代码图标
  icon = "github"       # ◀️ GitHub 图标
  icon = "link"         # ◀️ 链接图标
  icon = "folder"       # ◀️ 文件夹图标
```

:::

### 图标查找与使用

::: tip 🔍 图标资源
- 图标库：[Bootstrap Icons](https://icons.getbootstrap.com/)
- 使用方式：复制图标名称（不含 bi- 前缀）
- 示例：`bi-house` → 配置为 `icon = "house"`
:::

## 🔗 外部链接配置

### 配置外部链接菜单

通过 `params.external` 和 `params.target` 配置外部链接：

::: code-group

```toml [外部链接🔗]
[[main]]
  name = "⏱️ 时间轴"
  parent = "about"
  url = "https://wiki.xxdevops.cn/"
  weight = 32
  [main.params]
    icon = "clock"
    external = true      # ◀️ 标记为外部链接
    target = "_blank"    # ◀️ 在新窗口打开
```

```toml [外部链接参数说明📝]
[main.params]
  external = true       # ◀️ 添加外部链接标识（可选）
  target = "_blank"     # ◀️ 新窗口打开：_blank
                        # ◀️ 当前窗口打开：_self（默认）
```

:::

### 外部链接与内部链接对比

::: code-group

```toml [内部链接🏠]
[[main]]
  name = "👋 关于我"
  url = "/about/"
  weight = 1
  # ◀️ 无需 external 和 target 参数
```

```toml [外部链接🌐]
[[main]]
  name = "📊 网站统计"
  url = "https://umami.xxdevops.cn/"
  weight = 2
  [main.params]
    external = true
    target = "_blank"
```

:::

## 🔧 常见问题

### 菜单不显示或显示异常

::: code-group

```markdown [问题1：菜单不显示❌]
原因：identifier 重复或 parent 不匹配
解决：
1️⃣ 检查所有 identifier 是否唯一
2️⃣ 检查 parent 值是否与父菜单 identifier 一致
3️⃣ 重启开发服务器：make dev
```

```markdown [问题2：排序混乱❌]
原因：weight 值设置不当
解决：
1️⃣ 同级菜单使用连续的 weight 值
2️⃣ 建议以 10 为间隔，方便后续插入
3️⃣ 父菜单 weight 应小于子菜单
```

```markdown [问题3：图标不显示❌]
原因：icon 名称错误或格式问题
解决：
1️⃣ 访问 Bootstrap Icons 官网确认图标名称
2️⃣ 去掉 bi- 前缀，如 bi-house → house
3️⃣ 检查 params 缩进是否正确
```

:::

### 配置生效检查清单

```markdown
✅ 配置文件路径正确
✅ TOML 语法无误（使用在线 TOML 验证器检查）
✅ identifier 唯一不重复
✅ parent 与 identifier 匹配
✅ weight 数值合理
✅ 重启开发服务器后生效
```
