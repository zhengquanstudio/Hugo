## 📅 日历卡片概述

日历卡片以热力图形式展示文章发布记录，直观显示博主的更新频率和活跃程度。类似 GitHub Contributions 图表，颜色深浅代表当天发布文章的数量。

### ✨ 卡片特性

::: tip 🎨 设计特性
- 热力图展示：颜色深浅表示发布数量
- 年度概览：显示最近一年的发布记录
- 悬停提示：鼠标悬停显示具体日期和数量
- 暗色适配：自动适配暗色/亮色主题
:::

::: info ⚙️ 功能特性
- 自动统计：自动从文章数据生成
- 无需维护：数据自动更新，无需手动配置
- 独立开关：可随时启用/禁用
- 性能优化：数据缓存，快速渲染
:::

## 📝 开关配置

### 启用/禁用日历卡片

::: code-group

```toml [启用日历✅]
[params.homeSidebar.calendar]
    enabled = true   # ◀️ 显示日历卡片
```

```toml [禁用时历❌]
[params.homeSidebar.calendar]
    enabled = false  # ◀️ 隐藏日历卡片
```

:::

## 🎨 热力图说明

### 颜色规则

日历热力图使用颜色深浅表示文章发布数量：

::: code-group

```yaml [颜色说明🎨]
🟩 深绿色：发布 4+ 篇文章
🟨 中绿色：发布 3 篇文章
🟧 浅绿色：发布 2 篇文章
🟦 淡绿色：发布 1 篇文章
⬜ 灰色：无发布记录
```

```yaml [显示规则📋]
- 显示最近 365 天的发布记录
- 按周排列，每周一列
- 月份标签在顶部显示
- 悬停显示具体日期和文章数
```

:::

### 数据自动统计

::: info 🤖 自动统计机制
日历卡片数据完全自动统计，无需手动配置：

1️⃣ **数据来源**：自动读取所有文章的 `date` 字段
2️⃣ **统计范围**：最近 365 天的发布记录
3️⃣ **更新时机**：每次构建时重新统计
4️⃣ **缓存机制**：构建时生成缓存，提升访问速度
:::

## 🔧 常见问题

### 日历不显示或数据异常

::: code-group

```markdown [问题：日历不显示❌]
原因1：开关未启用
解决：检查 homepage.toml 中 enabled = true

原因2：无文章数据
解决：
1️⃣ 确保站点有已发布的文章
2️⃣ 检查文章 Front Matter 中 date 字段是否正确
3️⃣ 文章 date 格式应为："2026-03-02T10:00:00+08:00"

原因3：构建缓存问题
解决：执行 make clean 后重新构建
```

```markdown [问题：数据不准确❌]
原因：文章日期格式错误或时区问题
解决：
1️⃣ 统一使用 ISO 8601 格式：YYYY-MM-DDTHH:MM:SS+08:00
2️⃣ 检查时区设置是否正确
3️⃣ 确保文章已发布（draft: false）
```

:::

### 文章日期格式示例

::: code-group

```yaml [正确格式✅]
date: "2026-03-02T10:30:00+08:00"
date: "2026-03-02T10:30:00Z"
date: "2026-03-02"
```

```yaml [错误格式❌]
date: "2026/03/02"        # ◀️ 使用了斜杠
date: "03-02-2026"        # ◀️ 格式顺序错误
date: 2026-03-02          # ◀️ 缺少引号
```

:::
