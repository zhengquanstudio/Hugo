## ⏱️ 时间进度卡片概述

时间进度卡片以进度条形式展示年、月、周、日的时间流逝进度，提醒访客珍惜时间，也为站点增添动态元素。

### ✨ 卡片特性

::: tip 🎨 设计特性
- 四级进度：年、月、周、日进度一目了然
- 动态更新：实时计算当前进度百分比
- 进度条动画：平滑的进度条填充效果
- 时间显示：显示剩余时间和已用时间
:::

::: info ⚙️ 功能特性
- 自动计算：基于系统时间自动计算
- 无需维护：数据实时更新，无需配置
- 独立开关：可随时启用/禁用
- 农历支持：可选显示农历日期
:::

## 📝 开关配置

### 启用/禁用时间进度卡片

::: code-group

```toml [启用时间进度✅]
[params.homeSidebar.timeProgress]
    enabled = true   # ◀️ 显示时间进度卡片
```

```toml [禁用时间进度❌]
[params.homeSidebar.timeProgress]
    enabled = false  # ◀️ 隐藏时间进度卡片
```

:::

## 🎨 进度条说明

### 四级进度展示

时间进度卡片展示四个时间维度的进度：

::: code-group

```yaml [进度维度📊]

📆 月进度：显示当前月份已过去的时间百分比
📋 周进度：显示当前周已过去的时间百分比
⏰ 日进度：显示今天已过去的时间百分比
```

### 进度计算规则

::: info 🧮 计算规则
- **年进度**：已过天数 / 当年总天数（考虑闰年）
- **月进度**：已过天数 / 当月总天数
- **周进度**：已过天数 / 7 天（周一开始计算）
- **日进度**：已过小时数 / 24 小时
:::

## 🔧 常见问题

### 时间进度不显示

::: code-group

```markdown [问题：时间进度不显示❌]
原因1：开关未启用
解决：检查 homepage.toml 中 enabled = true

原因2：JavaScript 被禁用
解决：时间进度需要浏览器启用 JavaScript

原因3：时区问题
解决：检查系统时区设置是否正确
```

:::

### 进度计算不准确

::: code-group

```markdown [问题：进度不准确❌]
原因：浏览器缓存或时区差异
解决：
1️⃣ 清除浏览器缓存
2️⃣ 检查系统时区设置
3️⃣ 刷新页面重新计算

注意：
- 周进度从周一开始计算
- 年进度考虑闰年（366天）
- 进度计算基于访客本地时间
```

:::
