## 👤 关于我页面概述

Hugo-Teek 主题提供了一个精美的「关于我」页面模板，复刻张洪Heo风格设计，包含头像展示、个人介绍、技能展示、生涯时间线、心路历程等多个模块。

### ✨ 页面特性

::: tip 🎨 设计特性
- 顶部标签云：左右两侧展示个性化标签
- 头像展示：支持多种形状动画效果
- 介绍卡片：渐变卡片 + 文字卡片双栏布局
- 技能展示：无限滚动技能图标墙
- 生涯时间线：可视化职业发展历程
- 心路历程：卡片式故事展示
- 打赏功能：内置微信/支付宝打赏弹窗
- 留言板：集成评论系统
:::

::: info ⚙️ 配置特性
- YAML  frontmatter 配置，简单直观
- 所有内容通过参数配置，无需修改模板
- 支持自定义技能图标（需修改模板）
- 响应式设计，适配移动端
:::

::: warning 🔄 重要提示
- **修改文本信息后，需要重启项目才能生效**
- 重启项目，可以使用命令：`make dev` 或 `make build`
:::

## 📁 配置文件路径

关于我页面的配置分布在以下位置：

::: code-group

```yaml [页面内容配置📄]
# hugo-teek-site/content/about.md
# 关于页面主要内容配置
# 包括个人介绍、标签、生涯、心路历程等
```

```toml [作者信息配置📄]
# hugo-teek-site/config/teek-plugins/author.toml
# 作者基础信息（姓名、角色、座右铭、头像等）
```

```toml [博主卡片配置📄]
# hugo-teek-site/config/themes/hugo-teek/params.toml
# 博主卡片样式配置（头像形状、状态表情、社交链接等）
```

:::

### 配置优先级

```
配置合并顺序（优先级从低到高）：
1️⃣ config/themes/{theme}/params.toml（主题默认配置）
2️⃣ config/teek-plugins/author.toml（作者信息配置）
3️⃣ content/about.md（页面内容配置）
```

## 📝 基础页面配置

### 创建关于页面

在 `content/about.md` 创建关于页面：

::: code-group

```yaml [基础配置📝]
---
coverImg: https://mu.baimu.live/img/mn/
date: 2025-10-14T10:00:00+08:00
description: 关于 Hugo Teek 主题和我的个人介绍
draft: false
title: 关于
type: about
url: /about/
---
```

```yaml [配置说明📝]
---
coverImg: https://mu.baimu.live/img/mn/    # ◀️ 页面封面图
date: 2025-10-14T10:00:00+08:00            # ◀️ 页面创建日期
description: 关于 Hugo Teek 主题...       # ◀️ 页面描述（SEO用）
draft: false                               # ◀️ 是否为草稿
title: 关于                                # ◀️ 页面标题
type: about                                # ◀️ 页面类型，固定为 about
url: /about/                               # ◀️ 页面访问路径
---
```

:::

### Frontmatter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 页面标题，显示在浏览器标签页 |
| `type` | ✅ | 页面类型，必须设置为 `about` |
| `url` | ✅ | 页面访问路径，建议设置为 `/about/` |
| `coverImg` | ❌ | 页面封面图，显示在页面顶部 |
| `description` | ❌ | 页面描述，用于 SEO 优化 |
| `draft` | ❌ | 是否为草稿，`true` 时页面不发布 |

::: code-group

```toml [作者信息配置👤]
[params.author]
  # 作者姓名
  name = "余温Gueen"

  # 作者角色/职业
  role = "设计师、产品经理、独立开发者、博主"

  # 座右铭
  motto = "荆棘之路，劈风斩浪。"

  avatar = "https://img.xxdevops.cn/blog/avatar/yuwen_avatar.avif?w=300&h=300&fit=crop&fm=webp&q=80"

  email = ""

  # 作者个人网站或博客链接
  link = "https://xxdevops.cn"
```

```toml [配置说明📝]
[params.author]
  name = "..."       # ◀️ 作者姓名，显示在关于页面
```

## 🔧 高级自定义

### 自定义技能图标

技能图标需要在模板文件中添加，位于：

```
hugo-teek-site/themes/hugo-teek/layouts/about/single.html
```

在 `skills-track` 区域添加新的技能项：

```html
<div class="skill-item" data-name="你的技能名">
  <div class="skill-icon custom">自定义内容</div>
  <div class="skill-tooltip">你的技能名</div>
</div>
```

::: warning ⚠️ 注意事项
- 修改模板后需要重启开发服务器才能生效
- 建议复制一份到 `layouts/about/single.html` 进行修改，避免主题更新被覆盖
:::

### 页面样式自定义

如需自定义样式，可在 `assets/scss/custom.scss` 中添加：

```scss
/* 🎨 关于页面自定义样式 */
.about-page-v2 {
  /* 自定义样式 */
}
```

## 🔧 常见问题

### 页面不显示或显示异常

::: code-group

```markdown [问题1：页面404❌]
原因：页面未创建或 draft 为 true
解决：
1️⃣ 确认 content/about.md 文件存在
2️⃣ 检查 frontmatter 中 draft: false
3️⃣ 重启开发服务器：make dev
```

```markdown [问题2：头像不显示❌]
原因：头像 URL 错误或无法访问
解决：
1️⃣ 检查 author.toml 中的 avatar 链接
2️⃣ 检查 params.toml 中的 blogger.avatar 链接
3️⃣ 确保图片 URL 可公开访问
```

```markdown [问题3：打赏弹窗不显示❌]
原因：appreciation 配置未启用或二维码无效
解决：
1️⃣ 检查 params.toml 中 appreciation.enabled = true
2️⃣ 确认 alipayQR 和 wechatQR 图片链接有效
3️⃣ 检查 about.md 中 thanks 配置是否正确
```

:::

### 配置生效检查清单

```markdown
✅ content/about.md 文件存在且格式正确
✅ frontmatter 中 type: about 已设置
✅ frontmatter 中 draft: false
✅ author.toml 中作者信息已配置
✅ params.toml 中博主卡片已配置
✅ 图片 URL 可正常访问
✅ 重启开发服务器后生效
```
