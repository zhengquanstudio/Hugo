## 🎨 Banner区域配置概述

Hugo-Teek 主题的 Banner 区域是首页的视觉焦点，包含全屏背景图轮播、动态副标题打字机效果、波浪过渡动画等元素。Banner 配置采用 TOML 格式，提供丰富的自定义选项。

### ✨ Banner区域特性

::: tip 🎨 视觉特性
- 全屏背景轮播：支持多张壁纸自动切换
- 智能亮度检测：根据壁纸亮度自动调整导航栏样式
- 波浪过渡效果：底部 SVG 波浪动画
- 响应式适配：针对移动端和桌面端优化
:::

::: info ⚙️ 功能特性
- 三种工作模式：静态模式、API模式、混合模式
- 智能预加载：预测性预加载下一张图片
- 页面可见性优化：切换标签页时自动暂停
- 全屏壁纸模式：F11 全屏时纯净显示壁纸
:::

## 📁 配置文件路径

Hugo-Teek 采用分层配置结构，Banner 相关配置文件位于以下位置：

::: code-group

```toml [主配置文件📄]
# hugo-teek-site/config/themes/hugo-teek/params.toml
# 包含 heroBg、heroSubtitle 等核心配置
```

```yaml [副标题数据📄]
# hugo-teek-site/data/hero-subtitle.yaml
# 首页副标题打字机效果的文本列表
```

```json [背景图片数据📄]
# hugo-teek-site/data/hero-backgrounds.json
# 静态背景图片列表
```

:::

### 配置优先级

```
配置合并顺序（优先级从低到高）：
1️⃣ config/themes/{theme}/params.toml（主题默认配置）
2️⃣ data/hero-backgrounds.json（静态图片数据）
3️⃣ data/hero-subtitle.yaml（副标题文本数据）
4️⃣ 运行时通过管理后台修改的配置
```

## 🖼️ 首页背景配置

### 基础启用配置

最简单的背景配置，启用/禁用 Banner 背景：

::: code-group

```toml [启用背景🖼️]
[heroBg]
  enabled = true      # ◀️ 启用动态背景
  autoPlay = true     # ◀️ 自动轮播
  interval = 8000     # ◀️ 切换间隔（毫秒）
```

```toml [禁用背景🚫]
[heroBg]
  enabled = false     # ◀️ 禁用动态背景，显示纯色背景
```

:::

### 工作模式配置

Hugo-Teek 支持三种背景图片获取模式：

::: code-group

```toml [静态模式🏠]
[heroBg]
  enabled = true
  mode = "static"           # ◀️ 仅使用静态图片列表
  useStaticFirst = true     # ◀️ 优先使用静态列表
```

```toml [API模式🌐]
[heroBg]
  enabled = true
  mode = "api"              # ◀️ 仅从API获取图片
  apiEnabled = true         # ◀️ 启用API
  apiUrl = "https://imgapi.xxdevops.cn/api/images"
  apiCacheDuration = 3600   # ◀️ API缓存时长（秒）
```

```toml [混合模式🔀]
[heroBg]
  enabled = true
  mode = "hybrid"           # ◀️ 静态+API混合
  useStaticFirst = true     # ◀️ 优先使用静态图片
  apiEnabled = true
  apiUrl = "https://imgapi.xxdevops.cn/api/images"
```

:::

### 模式对比说明

| 模式 | 说明 | 适用场景 | 性能 |
|------|------|----------|------|
| `static` | 仅使用本地配置的图片 | 图片固定、追求性能 | ⭐⭐⭐⭐⭐ |
| `api` | 仅从远程API获取图片 | 图片动态更新 | ⭐⭐⭐ |
| `hybrid` | 静态图片优先，API作为补充 | 平衡性能与动态性 | ⭐⭐⭐⭐ |

在 `data/hero-backgrounds.json` 中配置静态背景图片：

::: code-group

```json [完整图片列表📝]
{
  "images": [

    {
      "url": "https://img.xxdevops.cn/blog/wallpaper/bg02.avif"
    },
    {
      "url": "https://img.xxdevops.cn/blog/wallpaper/bg03.avif"
    }

```

```json [带参数优化📝]
{
  "images": [
    {
      "url": "https://img.xxdevops.cn/blog/wallpaper/bg01.avif?w=1920&h=1080&q=85"
    },
    {
      "url": "https://img.xxdevops.cn/blog/wallpaper/bg02.avif?w=1920&h=1080&q=85"
    }
```

:::

### 图片格式建议

::: tip 📸 图片优化建议
- **格式**：推荐使用 WebP 或 AVIF 格式，体积更小
- **尺寸**：桌面端 1920×1080，移动端 1080×720
- **质量**：桌面端 85%，移动端 75%
- **体积**：单张图片建议不超过 500KB
:::

## 🎯 响应式图片质量配置

### 分设备类型配置

针对不同设备类型配置不同的图片质量参数：

::: code-group

```toml [响应式配置🎯]
[heroBg.quality]
  format = "webp"           # ◀️ 优先使用 WebP 格式
  fallbackFormat = "jpg"    # ◀️ 不支持 WebP 时降级为 JPG

  # ◀️ 移动端图片配置（<768px）
  [heroBg.quality.mobile]
    width = 1080
    height = 720
    quality = 75

  # ◀️ 桌面端图片配置（>=768px）
  [heroBg.quality.desktop]
    width = 1920
    height = 1080
    quality = 85
```

```toml [参数说明📝]
[heroBg.quality]
  format = "webp"           # ◀️ 图片格式：webp、jpg、png、avif
  fallbackFormat = "jpg"    # ◀️ 降级格式，用于旧浏览器

[heroBg.quality.mobile]
  width = 1080              # ◀️ 移动端图片宽度
  height = 720              # ◀️ 移动端图片高度
  quality = 75              # ◀️ 移动端压缩质量（1-100）

[heroBg.quality.desktop]
  width = 1920              # ◀️ 桌面端图片宽度
  height = 1080             # ◀️ 桌面端图片高度
  quality = 85              # ◀️ 桌面端压缩质量（1-100）
```

## ⚡ 性能优化配置

### 预加载与懒加载

配置图片预加载策略，优化首屏加载性能：

::: code-group

```toml [性能优化配置⚡]
[heroBg.performance]
  preloadCount = 3          # ◀️ 预加载前3张图片
  lazyLoadDelay = 500       # ◀️ 延迟500ms加载非关键图片
  predictivePreload = true  # ◀️ 智能预加载下一张
  pauseOnHidden = true      # ◀️ 页面隐藏时暂停切换和预加载
```

```toml [参数说明📝]
[heroBg.performance]
  preloadCount = 3          # ◀️ 预加载数量，建议 2-5 张
  lazyLoadDelay = 500       # ◀️ 懒加载延迟（毫秒）
  predictivePreload = true  # ◀️ 预测性预加载：true/false
  pauseOnHidden = true      # ◀️ 页面隐藏暂停：true/false
```

:::

### 性能优化建议

::: warning ⚠️ 注意事项
- `preloadCount` 不宜过大，建议 2-5 张，避免占用过多带宽
- `lazyLoadDelay` 建议 300-800ms，平衡加载速度和性能
- 开启 `pauseOnHidden` 可显著降低后台标签页的资源占用
- 预测性预加载会在切换前 500ms 自动加载下一张图片
:::

## ⌨️ 副标题打字机效果

### 基础配置

在 `config/themes/hugo-teek/params.toml` 中配置打字机效果参数：

::: code-group

```toml [打字机配置⌨️]
[heroSubtitle]
  typeSpeed = 90.0      # ◀️ 打字速度（毫秒/字符）
  deleteSpeed = 45.0    # ◀️ 删除速度（毫秒/字符）
  hold = 1800.0         # ◀️ 停留时间（毫秒）
  nextDelay = 600.0     # ◀️ 切换下一条的延迟（毫秒）
  shuffle = false       # ◀️ 是否随机打乱顺序
```

```toml [参数说明📝]
[heroSubtitle]
  typeSpeed = 90.0      # ◀️ 数值越小打字越快
  deleteSpeed = 45.0    # ◀️ 数值越小删除越快
  hold = 1800.0         # ◀️ 打完一条后停留时间
  nextDelay = 600.0     # ◀️ 删除后到下一条的间隔
  shuffle = false       # ◀️ true=随机顺序，false=顺序播放
```

:::

## 📋 完整配置示例

### 典型博客 Banner 配置

::: code-group

```toml [完整配置示例📋]
# ========================================
# 🖼️ 首页背景配置 (Hero Background)
# ========================================
[heroBg]
  # 启用动态背景
  enabled = true

  # 自动轮播配置
  autoPlay = true
  interval = 8000           # ◀️ 8秒切换（建议 8-12 秒）

  # 混合模式配置
  mode = "static"           # ◀️ static | hybrid | api
  useStaticFirst = true     # ◀️ 优先使用静态列表

  # API 配置（作为备份/补充）
  apiUrl = "https://imgapi.xxdevops.cn/api/images"
  apiEnabled = false        # ◀️ 启用 API 动态获取
  apiCacheDuration = 3600   # ◀️ API 缓存时长（秒）

  # 性能优化配置
  [heroBg.performance]
    preloadCount = 3
    lazyLoadDelay = 500
    predictivePreload = true
    pauseOnHidden = true

  # 图片质量配置
  [heroBg.quality]
    format = "webp"
    fallbackFormat = "jpg"

    [heroBg.quality.mobile]
      width = 1080
      height = 720
      quality = 75

    [heroBg.quality.desktop]
      width = 1920
      height = 1080
      quality = 85

# ========================================
# ⌨️ 首页副标题动画 (Hero Subtitle)
# ========================================
[heroSubtitle]
  typeSpeed = 90.0
  deleteSpeed = 45.0
  hold = 1800.0
  nextDelay = 600.0
  shuffle = false
```

## 🔧 常见问题

### 背景不显示或显示异常

::: code-group

```markdown [问题1：背景不显示❌]
原因：heroBg.enabled 设置为 false 或图片路径错误
解决：
1️⃣ 检查 config/themes/hugo-teek/params.toml 中 enabled 是否为 true
2️⃣ 检查 data/hero-backgrounds.json 中图片 URL 是否可访问
3️⃣ 重启开发服务器：make dev
```

```markdown [问题2：图片加载慢❌]
原因：图片体积过大或未启用预加载
解决：
1️⃣ 压缩图片至 500KB 以下
2️⃣ 启用 WebP/AVIF 格式
3️⃣ 调整 preloadCount 为 3-5
4️⃣ 配置 CDN 图片处理参数
```

```markdown [问题3：打字机效果不生效❌]
原因：hero-subtitle.yaml 格式错误或 items 为空
解决：
1️⃣ 检查 data/hero-subtitle.yaml 是否存在
2⃣ 确认 items 列表不为空
3️⃣ 检查 YAML 语法缩进是否正确
```

:::

### 配置生效检查清单

```markdown
✅ 配置文件路径正确
✅ TOML/YAML/JSON 语法无误
✅ 图片 URL 可正常访问
✅ heroBg.enabled 设置为 true
✅ 静态图片列表不为空
✅ 重启开发服务器后生效
```
