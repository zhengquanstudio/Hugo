## 🌐 友链页面配置概述

Hugo-Teek 主题内置了精美的友链页面模板，支持分组展示、头像显示、描述信息、链接统计等功能。友链配置采用 YAML 格式数据文件，通过 Hugo 的数据文件系统进行管理。

### ✨ 友链页面特性

::: tip 🎨 设计特性
- 分组展示：支持按分类分组展示友链
- 头像支持：自动获取或自定义友链头像
- 响应式布局：自适应网格布局，适配各种屏幕
- 悬停效果：精美的卡片悬停动画效果
:::

::: info ⚙️ 配置特性
- 数据驱动：通过 YAML 数据文件管理友链
- 灵活分组：支持自定义友链分类
- 参数扩展：支持自定义友链属性
- 排序控制：支持按权重排序友链
:::

## 📁 配置文件路径

Hugo-Teek 友链配置采用数据文件结构，配置文件位于以下位置：

::: code-group

```yaml [友链数据文件📄]
# hugo-teek-site/data/friends.yml
# 友链数据配置文件
# 包含所有友链信息
```

```markdown [友链页面文件📄]
# hugo-teek-site/content/about/friend-links.md
# 友链页面内容文件
# 定义页面元数据和基础内容
```

:::

### 配置优先级

```
配置加载顺序：
1️⃣ data/friends.yml（友链数据文件）
2️⃣ content/about/friend-links.md（页面内容文件）
3️⃣ 主题默认模板（layouts/_default/friends.html）
```

## 📝 友链数据配置

### 基础数据结构

最简单的友链配置，包含名称、链接和描述：

::: code-group

```yaml [基础友链配置📝]
friends:
  - name: "白木博客"
    url: "https://baimu.live"
    description: "记录生活，分享技术"
```

```yaml [配置说明📝]
friends:                          # ◀️ 友链表根节点
  - name: "白木博客"               # ◀️ 网站名称
    url: "https://baimu.live"     # ◀️ 网站链接
    description: "记录生活，分享技术" # ◀️ 网站描述
```

:::

### 友链配置字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 网站名称，显示在友链卡片上 |
| `url` | ✅ | 网站链接，点击跳转的目标地址 |
| `description` | ❌ | 网站描述，简短介绍网站内容 |
| `avatar` | ❌ | 头像链接，自定义友链头像 |
| `weight` | ❌ | 排序权重，数字越小越靠前 |

## 🌳 分组友链配置

### 创建分组结构

通过添加 `group` 字段实现友链分组展示：

::: code-group

```yaml [分组友链配置🌳]
friends:
  # 技术博客分组
  - group: "技术博客"
    weight: 1
    items:
      - name: "白木博客"
        url: "https://baimu.live"
        description: "记录生活，分享技术"
        avatar: "https://baimu.live/avatar.png"
        weight: 1
      
      - name: "Hugo官方"
        url: "https://gohugo.io"
        description: "Hugo静态网站生成器官方文档"
        weight: 2
  
  # 生活随笔分组
  - group: "生活随笔"
    weight: 2
    items:
      - name: "示例博客"
        url: "https://example.com"
        description: "一个示例博客网站"
        weight: 1
```

```yaml [层级关系说明📝]
🌐 友链页面
├── 📚 技术博客 (group)
│   ├── 白木博客
│   └── Hugo官方
└── 📝 生活随笔 (group)
    └── 示例博客
```

:::

### 分组配置最佳实践

::: warning ⚠️ 注意事项
- 每个分组的 `group` 名称必须唯一
- 分组内的 `weight` 控制该分组内友链的排序
- 分组的 `weight` 控制分组之间的排序顺序
- 建议合理分组，保持页面结构清晰
:::

## 🎨 头像配置

### 自定义友链头像

通过 `avatar` 字段配置友链头像：

::: code-group

```yaml [带头像的友链🎨]
friends:
  - name: "白木博客"
    url: "https://baimu.live"
    description: "记录生活，分享技术"
    avatar: "https://baimu.live/avatar.png"
    weight: 1
```

```yaml [头像获取方式🎨]
# 方式1：直接链接
avatar: "https://example.com/avatar.jpg"

# 方式2：使用 Gravatar
avatar: "https://www.gravatar.com/avatar/MD5_HASH"

# 方式3：使用 Cravatar（国内推荐）
avatar: "https://cravatar.cn/avatar/MD5_HASH"
```

:::

### 头像查找与使用

::: tip 🔍 头像资源
- 推荐使用 100x100 或 200x200 像素的正方形图片
- 支持格式：PNG、JPG、WebP
- 建议使用 HTTPS 链接确保安全性
- 如不提供头像，主题会显示默认占位图标
:::

## 🔧 常见问题

### 友链不显示或显示异常

::: code-group

```markdown [问题1：友链不显示❌]
原因：数据文件格式错误或路径不对
解决：
1️⃣ 检查 data/friends.yml 文件是否存在
2️⃣ 使用 YAML 验证器检查语法
3️⃣ 确保 friends 根节点存在
4️⃣ 重启开发服务器：make dev
```

```markdown [问题2：头像不显示❌]
原因：头像链接错误或无法访问
解决：
1️⃣ 检查 avatar 链接是否可访问
2️⃣ 确保使用 HTTPS 协议
3️⃣ 检查图片格式是否支持
4️⃣ 尝试使用其他图床
```

```markdown [问题3：排序混乱❌]
原因：weight 值设置不当
解决：
1️⃣ 分组间使用 weight 控制顺序
2️⃣ 分组内使用 weight 控制友链顺序
3️⃣ 建议以 10 为间隔，方便后续插入
```

:::

### 配置生效检查清单

```markdown
✅ 数据文件路径正确 (data/friends.yml)
✅ YAML 语法无误（使用在线 YAML 验证器检查）
✅ 页面文件使用 layout: "friends"
✅ 友链链接格式正确（包含 https://）
✅ 头像链接可正常访问
✅ 重启开发服务器后生效
```
