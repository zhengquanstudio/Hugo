## 🦶 底部栏配置概述

Hugo-Teek 主题提供高度可定制的站点底部栏（Footer），支持服务徽章展示、版权信息、ICP备案、不蒜子网站统计、个性化箴言等多种功能。

### ✨ 底部栏特性

::: tip 🎨 设计特性
- 服务徽章：展示技术栈和服务提供商图标
- 版权信息：自动计算年份范围，支持自定义后缀
- ICP备案：内置备案号展示，支持点击跳转查询
- 不蒜子统计：显示今日/昨日/本月访客数和访问量
- 箴言展示：支持展示个性化签名或座右铭
- 动物装饰：可爱的动物装饰图片
:::

::: info ⚙️ 配置特性
- 分层配置：内容与样式分离，便于主题切换
- 灵活开关：所有功能均提供独立启用/禁用选项
- 自动合并：通过 hugo-teek-tools 自动合并配置
- 多主题支持：不同主题可拥有独立的UI展示配置
:::

## 📁 配置文件路径

Hugo-Teek 采用分层配置结构，底部栏相关配置文件位于以下位置：

::: code-group

```toml [通用内容配置📄]
# hugo-teek-site/config/teek-plugins/footer.toml
# 包含版权、备案号、箴言等内容层配置
# 所有主题共用，推荐在此配置基础信息
```

```yaml [服务徽章配置📄]
# hugo-teek-site/data/footerServices.yaml
# 包含服务徽章列表（Hugo、Twikoo、CDN等）
# 控制底部服务图标展示
```

```toml [主题UI配置📄]
# hugo-teek-site/config/themes/hugo-teek/params.toml
# 包含 show_services 等UI控制配置
# 控制底部栏的显示元素
```

```toml [统计服务配置📄]
# hugo-teek-site/config/themes/hugo-teek/services.toml
# 包含不蒜子统计等服务配置
```

:::

### 配置优先级

```
配置合并顺序（优先级从低到高）：
1️⃣ config/themes/{theme}/params.toml（主题默认UI配置）
2️⃣ config/teek-plugins/footer.toml（通用内容配置）
3️⃣ data/footerServices.yaml（服务徽章数据）
4️⃣ config/themes/{theme}/services.toml（统计服务配置）
5️⃣ 运行时通过管理后台修改的配置
```

### 启用/禁用服务徽章

::: code-group

```toml [主题配置🎖️]
# config/themes/hugo-teek/params.toml
[footer]
    show_services = true      # ◀️ 设为 true 显示服务徽章，false 隐藏
```

```toml [配置说明🎖️]
[footer]
  show_services = true        # ◀️ 控制是否显示 footerServices.yaml 中配置的服务徽章
```

:::

## 📝 版权信息配置

### 版权信息配置

配置站点版权信息，支持起始年份和自定义后缀：

::: code-group

```toml [版权配置📝]
# config/teek-plugins/footer.toml
[params.footer]
  # 版权起始年份
  copyright_year_start = 2024.11

  # 版权后缀文本（显示在年份之后）
  copyright_suffix = "余温Gueen的博客"
```

```toml [配置说明📝]
[params.footer]
  copyright_year_start = 2024.11    # ◀️ 起始年份，支持小数表示月份
  copyright_suffix = "博客名称"      # ◀️ 显示格式：© 2024-2025 博客名称
```

:::

### 版权显示效果

| 配置项 | 说明 | 示例输出 |
|--------|------|----------|
| `copyright_year_start` | 起始年份 | `2024.11` 表示 2024年11月 |
| `copyright_suffix` | 版权后缀 | `余温Gueen的博客` |
| 自动计算 | 当前年份 | 自动显示为 `© 2024-2025` |

## 📋 ICP备案配置

### 启用ICP备案显示

中国大陆站点需要展示ICP备案信息，Hugo-Teek 提供内置支持：

::: code-group

```toml [ICP备案配置📋]
# config/teek-plugins/footer.toml
[params.footer.icp]
  # 是否启用 ICP 备案显示
  enabled = true

  # ICP 备案号
  number = "苏ICP备2024144209号"

  # ICP 备案查询链接
  url = "http://beian.miit.gov.cn/"
```

```toml [配置说明📋]
[params.footer.icp]
  enabled = true                              # ◀️ 总开关，设为 false 隐藏备案信息
  number = "苏ICP备2024144209号"              # ◀️ 你的备案号
  url = "http://beian.miit.gov.cn/"           # ◀️ 工信部备案查询链接
```

:::

### ICP备案最佳实践

::: warning ⚠️ 注意事项
- 备案号必须真实有效，与工信部登记信息一致
- 未备案站点请将 `enabled` 设为 `false`
- 备案信息变更后需及时更新配置
- 建议定期检查备案链接是否可正常访问
:::

## 📊 不蒜子统计配置

### 启用不蒜子统计

不蒜子统计可以显示站点的访问数据，包括今日/昨日/本月的访客数和访问量：

::: code-group

```toml [不蒜子统计配置📊]
# config/themes/hugo-teek/services.toml
[analytics]
  [analytics.busuanzi]
    enabled = true                    # ◀️ 总开关
    url = "https://busuanzi-eo.xxdevops.cn/busuanzi.eo.js"
    api = "https://busuanzi-eo.xxdevops.cn/api/count"
    type = "eo"                       # ◀️ EO 版本或 classic 版本
    prefix = "busuanzi"
    style = "comma"                   # ◀️ 数字格式：comma（千分位）| none
    pjax = false
    page_pv = true                    # ◀️ 显示页面浏览量
    site_pv = true                    # ◀️ 显示站点总浏览量
    site_uv = true                    # ◀️ 显示站点独立访客数
```

```toml [显示效果📊]
# 底部栏会显示以下统计信息：
今日访客数 1 | 今日访问量 387 | 昨日访客数 1 | 昨日访问量 1724 | 本月访客数 1 | 本月访问量 2110
```

:::

### 不蒜子统计字段说明

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| `enabled` | 总开关 | `true` / `false` |
| `type` | 版本类型 | `eo`（推荐）/ `classic` |
| `style` | 数字格式 | `comma`（千分位）/ `none` |
| `url` | JS文件地址 | 不蒜子服务URL |
| `api` | API端点 | 计数API地址 |

## 🔧 常见问题

### 底部信息不显示或显示异常

::: code-group

```markdown [问题1：服务徽章不显示❌]
原因：show_services 设为 false 或 footerServices.yaml 不存在
解决：
1️⃣ 检查 config/themes/hugo-teek/params.toml 中 show_services = true
2️⃣ 确认 data/footerServices.yaml 文件存在且格式正确
3️⃣ 重启开发服务器：make dev
```

```markdown [问题2：版权信息不显示❌]
原因：copyright_year_start 或 copyright_suffix 未配置
解决：
1️⃣ 检查 config/teek-plugins/footer.toml 是否存在
2️⃣ 确认 [params.footer] 节点配置正确
3️⃣ 重启开发服务器：make dev
```

```markdown [问题3：ICP备案不显示❌]
原因：enabled 设为 false 或配置未生效
解决：
1️⃣ 检查 [params.footer.icp] 下的 enabled = true
2️⃣ 确认 number 和 url 已正确填写
3️⃣ 运行 make build 重新合并配置
```

```markdown [问题4：不蒜子统计不显示❌]
原因：analytics.busuanzi.enabled 未启用
解决：
1️⃣ 检查 services.toml 中 [analytics.busuanzi] enabled = true
2️⃣ 确认 url 和 api 地址可正常访问
3️⃣ 检查网络连接是否正常加载不蒜子脚本
```

:::

### 配置生效检查清单

```markdown
✅ 配置文件路径正确
✅ TOML/YAML 语法无误
✅ 配置项位于正确的节点下
✅ 启用开关设为 true
✅ 运行 make dev 或 make build 重新合并配置
✅ 浏览器强制刷新（Ctrl+F5）清除缓存
```
