## 💰 赞赏图配置概述

Hugo-Teek 主题内置了优雅的赞赏功能，支持微信支付和支付宝两种打赏方式。赞赏功能采用可折叠设计，点击按钮后展开显示二维码，既美观又不占用过多页面空间。

### ✨ 赞赏功能特性

::: tip 🎨 设计特性
- 折叠式设计：点击展开，节省页面空间
- 双支付方式：支持微信支付和支付宝
- 精美动效：按钮悬停动画、二维码浮动效果
- 响应式布局：移动端自动适配
:::

::: info ⚙️ 配置特性
- 独立开关：可单独启用或禁用赞赏功能
- 图片灵活：支持本地图片或外部链接
- 格式兼容：支持 AVIF、WebP、PNG、JPG 等格式
- 主题隔离：不同主题可拥有独立赞赏配置
:::

## 📁 配置文件路径

Hugo-Teek 采用分层配置结构，赞赏图配置文件位于以下位置：

::: code-group

```toml [主题特定配置📄]
# hugo-teek-site/config/themes/hugo-teek/params.toml
# 仅当前主题使用的赞赏配置
# 推荐在此文件中进行赞赏配置
```

```toml [通用配置📄]
# hugo-teek-site/config/teek-plugins/params.toml
# 所有主题共用的赞赏配置
# 用于覆盖主题默认配置
```

:::

### 配置优先级

```
配置合并顺序（优先级从低到高）：
1️⃣ config/themes/{theme}/params.toml（主题默认配置）
2️⃣ config/teek-plugins/params.toml（通用配置）
3️⃣ 运行时通过管理后台修改的配置
```

## 📝 基础赞赏配置

### 启用赞赏功能

最简单的赞赏配置，只需启用并设置二维码图片地址：

::: code-group

```toml [基础配置📝]
# ========================================
# 打赏配置 (Appreciation)
# ========================================
[appreciation]
    enabled = true  # ◀️ 启用打赏支持功能
    alipayQR = "https://your-domain.com/alipay.avif"   # ◀️ 支付宝二维码
    wechatQR = "https://your-domain.com/wechat.avif"   # ◀️ 微信二维码
```

```toml [配置说明📝]
[appreciation]
    enabled = true          # ◀️ 总开关：true 启用，false 禁用
    alipayQR = "图片URL"     # ◀️ 支付宝收款二维码图片地址
    wechatQR = "图片URL"     # ◀️ 微信收款二维码图片地址
```

:::

### 赞赏配置字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `enabled` | ✅ | 赞赏功能总开关，`true` 启用，`false` 禁用 |
| `alipayQR` | ❌ | 支付宝收款二维码图片 URL，留空则不显示支付宝选项 |
| `wechatQR` | ❌ | 微信收款二维码图片 URL，留空则不显示微信选项 |

## 🖼️ 二维码图片配置

### 使用外部图片链接

推荐使用图床或 CDN 托管二维码图片，加载速度快且便于管理：

::: code-group

```toml [外部图片链接🌐]
[appreciation]
    enabled = true
    alipayQR = "https://img.xxdevops.cn/blog/appreciation_code/zhifubao.avif"
    wechatQR = "https://img.xxdevops.cn/blog/appreciation_code/wechat.avif"
```

```toml [配置说明📝]
[appreciation]
    enabled = true
    # ◀️ 使用 HTTPS 协议的图片链接
    alipayQR = "https://your-cdn.com/alipay.avif"
    # ◀️ 支持 AVIF、WebP、PNG、JPG 等格式
    wechatQR = "https://your-cdn.com/wechat.webp"
```

:::

### 使用本地图片

也可以将二维码图片放在项目的 `static` 目录下：

::: code-group

```toml [本地图片🏠]
[appreciation]
    enabled = true
    alipayQR = "/images/appreciation/alipay.png"
    wechatQR = "/images/appreciation/wechat.png"
```

```bash [目录结构📁]
hugo-teek-site/
└── static/
    └── images/
        └── appreciation/
            ├── alipay.png    # ◀️ 支付宝二维码
            └── wechat.png    # ◀️ 微信二维码
```

:::

### 图片格式建议

::: tip 🖼️ 图片优化建议
- **推荐格式**：AVIF > WebP > PNG > JPG
- **推荐尺寸**：400x400 像素（正方形）
- **文件大小**：建议控制在 50KB 以内
- **背景**：建议使用白色背景，确保扫码识别率
:::

## 🔍 常见问题

### 二维码不显示或显示异常

::: code-group

```markdown [问题1：二维码不显示❌]
原因：图片链接错误或图片不存在
解决：
1️⃣ 检查图片 URL 是否正确
2️⃣ 确认图片文件存在且可访问
3️⃣ 检查图片格式是否支持（AVIF、WebP、PNG、JPG）
4️⃣ 确保图片使用 HTTPS 协议（如使用外部链接）
```

```markdown [问题2：无法扫码识别❌]
原因：二维码图片被压缩或尺寸不合适
解决：
1️⃣ 使用原始尺寸二维码图片
2️⃣ 确保二维码周围有白色边距（quiet zone）
3️⃣ 检查图片是否被 CSS 裁剪或变形
4️⃣ 建议二维码尺寸不小于 300x300 像素
```

```markdown [问题3：赞赏按钮不显示❌]
原因：enabled 设置为 false 或配置未生效
解决：
1️⃣ 检查 enabled = true 是否设置正确
2️⃣ 确认配置文件路径正确
3️⃣ 重启开发服务器：make dev
4️⃣ 清除浏览器缓存后刷新页面
```

:::

### 配置生效检查清单

```markdown
✅ 配置文件路径正确
✅ TOML 语法无误（使用在线 TOML 验证器检查）
✅ enabled 设置为 true
✅ 二维码图片 URL 可访问
✅ 图片格式为 AVIF、WebP、PNG 或 JPG
✅ 重启开发服务器后生效
```

## 💡 最佳实践

### 二维码生成建议

::: tip 🎯 生成高质量二维码
1. **使用官方工具**：通过微信/支付宝官方渠道生成收款二维码
2. **保存高清原图**：下载时选择高清版本，避免压缩
3. **格式转换**：使用 AVIF 或 WebP 格式，体积更小画质更好
4. **CDN 加速**：将图片上传至 CDN，提升加载速度
:::

### 安全建议

::: warning ⚠️ 安全注意事项
- 定期检查二维码是否过期或被篡改
- 避免在公共仓库提交真实的收款二维码
- 建议使用图床或 CDN，便于随时更换
- 测试时可以使用测试二维码，上线前替换为正式二维码
:::
