## 🔍 站点监控页面概述

Hugo-Teek 主题支持接入第三方站点监控服务，实时展示站点的运行状态、可用性、响应时间等关键指标。通过监控页面，访客可以了解站点的服务健康状况。

### ✨ 监控功能特性

::: tip 🎨 展示特性
- 🚦 实时状态：展示站点各项服务的实时运行状态
- 📊 可用性统计：显示服务在线率、响应时间等指标
- 📈 历史记录：查看服务的历史运行状态和故障记录
- 🔔 故障告警：服务异常时及时通知
:::

::: info ⚙️ 配置特性
- 🔗 外部集成：支持Uptime Kuma、UptimeRobot等监控服务
- 🌐 独立页面：可嵌入监控面板或链接到外部监控页面
- 📱 响应式设计：适配各种屏幕尺寸
- 🎨 状态标识：直观的状态颜色标识（绿色/黄色/红色）
:::

## 🔗 导航栏监控入口配置

### 添加站点监控菜单项

在导航栏添加站点监控入口，链接到外部监控面板：

::: code-group

```toml [下拉菜单中的监控入口📄]
# config/teek-plugins/menus.toml
[[main]]
  name = "🔍 站点监控"
  parent = "about"
  url = "https://status.xxdevops.cn/status/dashboard"
  weight = 34.0
  [main.params]
    icon = "activity"
    external = true       # ◀️ 标记为外部链接
    target = "_blank"     # ◀️ 在新窗口打开
```

```toml [独立监控菜单项📄]
[[main]]
  name = "🔍 站点监控"
  url = "https://status.example.com/"
  weight = 10
  [main.params]
    icon = "activity"
    external = true
    target = "_blank"
```

```toml [配置说明📝]
[[main]]
  name = "🔍 站点监控"           # ◀️ 菜单显示名称
  parent = "about"               # ◀️ 父菜单标识符（放在【关于】下拉菜单下）
  url = "https://status..."      # ◀️ 监控面板地址
  weight = 34.0                  # ◀️ 排序权重
  [main.params]
    icon = "activity"            # ◀️ Bootstrap Icons 图标名称
    external = true              # ◀️ 标记为外部链接，显示外链图标
    target = "_blank"            # ◀️ 在新窗口打开监控页面
```

:::

## 🖥️ Uptime Kuma 监控配置

### Uptime Kuma 简介

Uptime Kuma 是一款开源的站点监控工具，支持监控HTTP/HTTPS、TCP、Ping等多种服务类型，提供美观的状态页面。

::: tip 🌟 Uptime Kuma 特点
- 🆓 开源免费，自托管部署
- 📊 支持多种监控类型（HTTP/TCP/Ping/DNS等）
- 🔔 多种通知渠道（邮件、Webhook、Telegram等）
- 🎨 可自定义状态页面样式
- 📱 响应式设计，支持移动端
:::

### 部署 Uptime Kuma

::: code-group

```bash [Docker部署🐳]
# 使用 Docker 快速部署 Uptime Kuma
docker run -d \
  --restart=always \
  -p 3001:3001 \
  -v uptime-kuma:/app/data \
  --name uptime-kuma \
  louislam/uptime-kuma:1
```

```bash [Docker Compose部署🐳]
# docker-compose.yml
version: '3.8'
services:
  uptime-kuma:
    image: louislam/uptime-kuma:1
    container_name: uptime-kuma
    volumes:
      - ./uptime-kuma-data:/app/data
    ports:
      - "3001:3001"
    restart: always
```

```bash [访问服务🌐]
# 部署完成后访问
http://localhost:3001
# 或你的服务器地址
http://your-server-ip:3001
```

:::

### 配置监控项

::: code-group

```yaml [HTTP监控配置📊]
# 在 Uptime Kuma 后台添加监控项
监控类型: HTTP(s)
名称: 我的博客首页
URL: https://xxdevops.cn/
心跳间隔: 60秒           # ◀️ 检查频率
重试次数: 3次            # ◀️ 失败重试次数
超时时间: 48秒           # ◀️ 请求超时时间
```

```yaml [API监控配置📊]
监控类型: HTTP(s)
名称: API接口
URL: https://api.xxdevops.cn/health
心跳间隔: 30秒
期望状态码: 200
```

```yaml [Ping监控配置📊]
监控类型: Ping
名称: 服务器连通性
主机名: xxdevops.cn
心跳间隔: 60秒
```

:::

### 获取状态页面链接

::: code-group

```yaml [创建状态页面📋]
1️⃣ 登录 Uptime Kuma 管理后台
2️⃣ 进入「状态页面」菜单
3️⃣ 点击「新建状态页面」
4️⃣ 配置页面标题、描述、Logo等
5️⃣ 选择要展示的监控项
6️⃣ 保存并获取公开访问链接
```

```yaml [状态页面URL格式🔗]
# 默认状态页面地址
https://status.xxdevops.cn/status/default

# 自定义slug的状态页面
https://status.xxdevops.cn/status/dashboard
https://status.xxdevops.cn/status/blog
```

## 📊 UptimeRobot 监控配置

### UptimeRobot 简介

UptimeRobot 是一款流行的站点监控服务，提供免费和付费版本，支持50个免费监控项。

::: tip 🌟 UptimeRobot 特点
- 🆓 免费版支持50个监控项
- ⏱️ 5分钟监控频率（免费版）
- 🔔 多种通知方式
- 📊 公开状态页面
- 🔌 REST API支持
:::

### 配置公开状态页面

::: code-group

```yaml [创建状态页面📋]
1️⃣ 登录 UptimeRobot 控制台
2️⃣ 进入「My Settings」->「Public Status Pages」
3️⃣ 点击「Add Public Status Page」
4️⃣ 配置页面信息：
   - Friendly Name: 站点状态
   - Monitor: 选择要展示的监控项
   - Custom Domain: 可选，绑定自定义域名
5️⃣ 保存并获取公开链接
```

```yaml [状态页面链接🔗]
# UptimeRobot 公开状态页面链接格式
https://stats.uptimerobot.com/你的页面ID

# 示例
https://stats.uptimerobot.com/xxxxx12345
```

:::

## 🔧 常见问题

### 监控页面无法访问

::: code-group

```markdown [问题1：监控页面404❌]
原因：监控服务未部署或链接错误
解决：
1️⃣ 确认 Uptime Kuma/UptimeRobot 已正确部署
2️⃣ 检查监控服务是否正常运行
3️⃣ 验证状态页面链接是否正确
4️⃣ 确认监控服务防火墙设置允许访问
```

```markdown [问题2：iframe无法加载❌]
原因：X-Frame-Options 限制或跨域问题
解决：
1️⃣ 检查监控服务是否允许嵌入iframe
2️⃣ 配置监控服务的 CORS 设置
3️⃣ 考虑改为直接链接到外部页面
4️⃣ 使用新窗口打开监控页面
```

```markdown [问题3：导航菜单不显示❌]
原因：导航栏配置错误或缓存问题
解决：
1️⃣ 检查 menus.toml 语法是否正确
2️⃣ 确认 parent 标识符存在且正确
3️⃣ 重启 Hugo 开发服务器
4️⃣ 清除浏览器缓存
```

:::

### Docker Compose 完整示例

::: code-group

```yaml [docker-compose.yml🐳]
version: '3.8'

services:
  uptime-kuma:
    image: louislam/uptime-kuma:1
    container_name: uptime-kuma
    volumes:
      - ./uptime-kuma-data:/app/data
    ports:
      - "3001:3001"
    restart: always
    networks:
      - monitoring

  # 可选：配合 Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: nginx
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    ports:
      - "80:80"
      - "443:443"
    restart: always
    networks:
      - monitoring

networks:
  monitoring:
    driver: bridge
```

```nginx [nginx.conf🌐]
server {
    listen 80;
    server_name status.xxdevops.cn;
    
    location / {
        proxy_pass http://uptime-kuma:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
