## 🧭 网站导航页面概述

Hugo-Teek 主题内置了精美的网站导航页面功能，支持分组展示、卡片式布局、徽章标记等特性。通过数据驱动的方式，您可以轻松管理和展示常用网站链接。

### ✨ 导航页面特性

::: tip 🎨 设计特性
- 📦 卡片式布局：响应式网格展示网站链接
- 🏷️ 徽章标记：支持为网站添加标签徽章
- 🖼️ 图标展示：自动加载网站图标或自定义图片
- 📱 响应式设计：适配各种屏幕尺寸
:::

::: info ⚙️ 配置特性
- 📊 数据驱动：通过 JSON 文件管理网站数据
- 🗂️ 分组管理：支持多分组分类展示
- 🔗 新窗口打开：外部链接默认在新窗口打开
- 🎭 骨架屏加载：图片加载时显示骨架屏效果
:::

## 📊 数据配置（websites.json）

### 数据结构说明

网站导航数据采用 JSON 格式存储，支持分组管理和丰富的网站属性：

::: code-group

```json [完整数据结构📊]
{
  "groups": [
    {
      "title": "分组标题",
      "sites": [
        {
          "name": "网站名称",
          "desc": "网站描述",
          "img": "图标URL地址",
          "link": "网站链接地址",
          "badge": "徽章文本",
          "badgeType": "徽章类型"
        }
      ]
    }
  ]
}
```

```yaml [字段说明📝]
groups:                    # ◀️ 分组数组
  - title: 分组标题         # ◀️ 分组显示名称
    sites:                 # ◀️ 网站数组
      - name: 网站名称      # ◀️ 网站显示名称（必填）
        desc: 网站描述      # ◀️ 网站简短描述
        img: 图标URL        # ◀️ 网站图标或Logo地址
        link: 网站链接      # ◀️ 网站跳转链接（必填）
        badge: 徽章文本     # ◀️ 右上角徽章文字
        badgeType: 类型     # ◀️ 徽章样式类型
```

:::

### 网站字段详细说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 网站显示名称 |
| `link` | ✅ | 网站跳转链接，支持相对/绝对路径 |
| `desc` | ❌ | 网站描述文字，显示在卡片下方 |
| `img` | ❌ | 网站图标/Logo URL，支持各种图片格式 |
| `badge` | ❌ | 徽章文字，显示在卡片右上角 |
| `badgeType` | ❌ | 徽章样式类型：`info`、`tip`、`warning`、`danger` |

## 📝 页面创建（websites.md）

### Front Matter 配置

创建网站导航页面需要设置特定的 Front Matter 参数：

::: code-group

```yaml [页面配置📝]
---
coverImg: https://mu.baimu.live/img/mn/        # ◀️ 封面图片
date: 2025-11-17T20:00:00+08:00                # ◀️ 创建日期
description: 精选优质网站导航                   # ◀️ 页面描述
draft: false                                   # ◀️ 是否草稿
lastmod: 2025-11-17T22:30:00+08:00             # ◀️ 最后修改时间
layout: websites                               # ◀️ 布局模板（必填）
title: 网站导航                                # ◀️ 页面标题
type: websites                                 # ◀️ 页面类型（必填）
url: /about/websites/                          # ◀️ 自定义URL路径
---
```

```yaml [字段说明📄]
layout: websites    # ◀️ 必须设置为 websites，使用主题内置布局
type: websites      # ◀️ 必须设置为 websites，用于样式识别
url: /about/websites/  # ◀️ 自定义访问路径，与导航栏配置保持一致
```

:::

### 使用 navCard Shortcode

在页面内容中使用 `navCard` Shortcode 渲染分组：

::: code-group

```markdown [基础用法🎯]
# 我的站点

{{< navCard group="我的站点" >}}

# VitePress网站

{{< navCard group="VitePress网站" >}}
```

```markdown [完整示例📋]
# 我的站点

{{< navCard group="我的站点" >}}

# VitePress网站

{{< navCard group="VitePress网站" >}}

# 静态站点构建器(SSG)

{{< navCard group="静态站点构建器(SSG)" >}}

# Docusaurus

{{< navCard group="Docusaurus" >}}

# 搜索引擎

{{< navCard group="搜索引擎" >}}

# 工具

{{< navCard group="工具" >}}

# 图床软件

{{< navCard group="图床软件" >}}

# 壁纸网站

{{< navCard group="壁纸网站" >}}
```

:::

### Shortcode 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `group` | ✅ | 对应 `websites.json` 中的分组标题 |

::: warning ⚠️ 注意事项
- `group` 参数必须与 `websites.json` 中的 `title` 完全一致
- 如果分组不存在，页面会显示提示信息
- 每个分组可以多次使用，但通常每个分组只使用一次
:::

## 🔗 导航栏菜单配置

### 添加导航栏入口

在导航栏配置文件中添加网站导航页面入口：

::: code-group

```toml [导航栏配置📄]
# hugo-teek-site/config/teek-plugins/menus.toml

[[main]]
  name = "🌐 网站导航"
  parent = "about"
  url = "/about/websites/"
  weight = 29.0
  [main.params]
    icon = "compass"
```

```toml [配置说明📝]
[[main]]
  name = "🌐 网站导航"       # ◀️ 菜单显示名称
  parent = "about"           # ◀️ 父菜单标识符（放在【关于】下拉菜单下）
  url = "/about/websites/"   # ◀️ 链接地址，与页面 url 保持一致
  weight = 29.0              # ◀️ 排序权重
  [main.params]
    icon = "compass"         # ◀️ Bootstrap Icons 图标名称
```

:::

### 独立导航项配置

如果需要将网站导航作为独立的一级菜单：

::: code-group

```toml [独立导航项🎯]
[[main]]
  name = "🌐 网站导航"
  url = "/about/websites/"
  weight = 6
  [main.params]
    icon = "compass"
```

```toml [下拉菜单中的导航项📂]
# 父菜单
[[main]]
  identifier = "resources"
  name = "📦 资源"
  url = "#"
  weight = 5

# 子菜单 - 网站导航
[[main]]
  name = "🌐 网站导航"
  parent = "resources"
  url = "/about/websites/"
  weight = 1
  [main.params]
    icon = "compass"
```

:::

网站导航卡片支持多种徽章样式：

::: code-group

```json [徽章类型示例🏷️]
{
  "sites": [
    {
```

## 🔧 完整配置流程

### 步骤一：配置数据文件

1. 创建或编辑 `data/websites.json` 文件
2. 按照 JSON 格式定义分组和网站数据
3. 验证 JSON 语法正确性

### 步骤二：创建页面文件

1. 创建 `content/websites.md` 文件
2. 配置 Front Matter（设置 `layout: websites` 和 `type: websites`）
3. 使用 `navCard` Shortcode 引用分组

### 步骤三：配置导航栏

1. 编辑 `config/teek-plugins/menus.toml`
2. 添加网站导航菜单项
3. 设置正确的 `url` 路径

## ❓ 常见问题

### 页面显示空白或分组不显示

::: code-group

```markdown [问题1：分组不显示❌]
原因：navCard 的 group 参数与 websites.json 中的 title 不匹配
解决：
1️⃣ 检查 group 参数是否与 title 完全一致（包括空格）
2️⃣ 检查 JSON 语法是否正确
3️⃣ 重启 Hugo 开发服务器
```

```markdown [问题2：图片不显示❌]
原因：图片地址错误或无法访问
解决：
1️⃣ 检查 img 字段的 URL 是否正确
2️⃣ 确保图片地址支持跨域访问
3️⃣ 使用可靠的图床或 CDN
```

```markdown [问题3：导航菜单不显示❌]
原因：导航栏配置错误或缓存问题
解决：
1️⃣ 检查 menus.toml 语法是否正确
2️⃣ 确认 url 与页面 url 一致
3️⃣ 重启开发服务器
```

:::

### 配置检查清单

```markdown

✅ websites.md 设置了 layout: websites 和 type: websites
✅ 导航栏配置的 url 与页面 url 一致
```
